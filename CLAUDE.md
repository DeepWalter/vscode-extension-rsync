This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run compile          # TypeScript compilation (tsc -p ./)
npm run watch            # Watch mode compilation
npm run lint             # ESLint on src/
npm run test             # Run extension tests via vscode-test
npm run pretest          # Compile + lint before test
npm run vscode:prepublish  # Compile (used by vsce package/publish)
```

- Press **F5** in VS Code to launch the Extension Development Host for manual testing.
- Test files live in `src/test/` and compile to `out/test/`. The test runner config is in [.vscode-test.mjs](.vscode-test.mjs).

## Architecture

This is a standard VS Code extension scaffold (generated by `yo code`). No custom activation events — the extension activates on the `rsync.helloWorld` command.

- **[src/extension.ts](src/extension.ts)** — Entry point. `activate()` registers the `rsync.helloWorld` command (shows an info message) and pushes the disposable to `context.subscriptions`. `deactivate()` is a no-op.
- **[package.json](package.json)** — Declares the single contributed command `rsync.helloWorld`, the `main` entry (`./out/extension.js`), and engine requirement `^1.120.0`.

Source TypeScript compiles to `out/` (ES2022 target, Node16 modules, strict mode enabled). Files excluded from the published `.vsix` are listed in [.vscodeignore](.vscodeignore).
